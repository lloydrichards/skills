# Reference profile

**Reader job:** Retrieve precise information quickly and repeatedly.

**Reasoning chain:** Orientation → taxonomy → entries → exceptions and constraints → related material.

Optimize for lookup rather than linear reading. Define the organizing principle early. Keep entry structures consistent, terminology exact, and anchors stable. Put constraints and exceptions beside the entry they qualify. Use tables only for genuinely repeated fields.

Avoid narrative padding, excessive disclosure, inconsistent entry shapes, and conclusions that force a linear reading path.
