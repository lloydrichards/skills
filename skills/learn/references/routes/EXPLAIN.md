# Explain route

Use when the learner needs a bounded mental model or misconception repaired.

## Sequence

1. Name what the learner should be able to explain or predict.
2. Elicit the current model or an initial prediction when useful.
3. Introduce the smallest accurate model using one concrete representation.
4. Trace a worked case through cause and effect.
5. Ask for a committed prediction on a nearby case before reveal.
6. Give explanatory feedback and repair the model.
7. End with a distinct transfer prompt.

Prefer Markdown when prose, code, or a static diagram carries the model. Use HTML when manipulation, staged reveal, or a dynamic trace materially reduces cognitive effort.

This route may support `introduced`, `practiced`, or—with successful independent transfer—`demonstrated`. It cannot establish `retained` in the same session that introduced the model.
