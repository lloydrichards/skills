# Review route

Use after a meaningful delay to test storage strength and decide the next check.

## Sequence

1. Select one or a few mission-relevant capabilities due for a qualitative check.
2. Ask for retrieval, prediction, diagnosis, or application before restudy.
3. Preserve the initial response.
4. Give corrective feedback and selective restudy only after commitment.
5. If appropriate, use a nearby case to distinguish recall from flexible application.
6. Update evidence and schedule one next check.

A successful delayed check may support `retained`. State the delay and evidence rather than applying a universal interval.

If retrieval fails, do not automatically demote the capability. Record the gap, repair it, and plan corrective practice. Avoid massed repetition disguised as review.
