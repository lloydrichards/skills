# Reference route

Use when the learner needs to retrieve syntax, terminology, mappings, invariants, algorithms, or procedures later.

Optimize for lookup rather than narrative completion:

- use stable names and headings;
- put the answer close to the likely query;
- prefer tables for exact repeated mappings;
- include minimal examples and boundary cases;
- keep terminology consistent with the workspace glossary if one exists;
- make Markdown and print the defaults unless interaction materially improves retrieval.

Reference artifacts do not establish capability state merely by existing or being read. Creating a reference may itself produce evidence only when the learner constructs or compresses it and that action matches the capability.
